نظام المصادقة الثنائي العامل (2FA) هو نظام أمان يعزز أمان الحسابات من خلال تطلب مصادقة إضافية إلى جانب كلمة المرور. هنا تصميم للنظام:

### 1. تسجيل الدخول الأولي
- **إدخال بيانات**: المستخدم يدخل اسم المستخدم وكلمة المرور.
- **التحقق الأولي**: يتم التحقق من صحة اسم المستخدم وكلمة المرور مقابل البيانات المخزنة في قاعدة البيانات.

### 2. إضافة مصادقة ثنائية العامل
- **اختيار طريقة 2FA**: بعد الدخول بنجاح، يُطلب من المستخدم اختيار طريقة للمصادقة الثنائية مثل:
  - رمز مرسل عبر رسالة نصية (SMS).
  - رمز تطبيقات المصادقة مثل Google Authenticator.
  - البريد الإلكتروني.
- **تكوين 2FA**: بعد اختيار الطريقة، يconfigure المستخدم إعدادات 2FA الخاصة به، مثل ربط رقم الهاتف أو تطبيق المصادقة.

### 3. عملية المصادقة
- **التحقق الأول**: عند محاولة الدخول، يُطلب من المستخدم إدخال اسم المستخدم وكلمة المرور.
- **إرسال رمز 2FA**: بعد التحقق الأولي الناجح، يُرسل رمز 2FA إلى المستخدم عبر الطريقة المُختارة.
- **التحقق الثاني**: يجب على المستخدم إدخال رمز 2FA الذي تلقاه لcompletion المصادقة.

### إجراءات الاستعادة
- **النسخ الاحتياطي للبيانات**: يجب على النظام تخزين نسخة احتياطية من بيانات المستخدمين، بما في ذلك إعدادات 2FA، في مكان آمن.
- **استعادة الحساب**: في حالة فقدان بيانات المستخدم أو عجز عن الوصول إلى حسابه، يمكن للمستخدم طلب استعادة الحساب من خلال：
  - إرسال رمز إلى البريد الإلكتروني المسجل أو رقم الهاتف.
  - إجراء التحقق عبر أسئلة أمان مسبقة التكوين.
  - استخدام ميزة استعادة خاصة إذا كان تطبيق 2FA ي지원ها.

### تدفق البيانات
1. **إدخال بيانات المستخدم**: المستخدم يدخل بياناته.
2. **التحقق الأولي**: يتم التحقق من بيانات الدخول.
3. **إرسال رمز 2FA**: يُرسل رمز إلى المستخدم عبر الطريقة المختارة.
4. **التحقق الثاني**: يتم التحقق من رمز 2FA الذي أدخله المستخدم.
5. **وصول آمن**: بعد المصادقة الناجحة، يمكن للمستخدم الوصول إلى حسابه وتطبيقاته.

### ميزات إضافية
- **تذكر الجهاز**: خيار لتذكر الجهاز أو المتصفح لمدة معينة لتجنب طلب 2FA في كل مرة.
- **استخدام بصمة الإصبع أو الوجه**: إذا كان الجهاز يدعمها، يمكن استخدامها كطريقة 2FA بديلة.
- **إشعارات الأمان**: إرسال إشعارات إلى المستخدم عند كل محاولة دخول غير معتادة.

## **2. تدفق البيانات**
### **2.1 تسجيل الدخول الأولي**
1. المستخدم يدخل اسم المستخدم (`username`) وكلمة المرور (`password`).
2. الخادم يتحقق من صحة البيانات باستخدام خوارزمية تشفير قوية (مثل `bcrypt`).
3. إذا كانت البيانات صحيحة، يتم إنشاء جلسة (`session`) للمستخدم.

#### **كود التحقق من كلمة المرور (Node.js + bcrypt)**
```javascript
const bcrypt = require('bcrypt');
const usersDB = require('./database').users; // افتراض وجود قاعدة بيانات

async function verifyPassword(username, password) {
    const user = await usersDB.findOne({ username });
    if (!user) return false;

    const match = await bcrypt.compare(password, user.passwordHash);
    return match;
}
```

### **2.2 إعداد المصادقة الثنائية (2FA)**
#### **خيارات 2FA المدعومة**
1. **رمز عبر SMS** (باستخدام Twilio).
2. **رمز عبر البريد الإلكتروني** (باستخدام SendGrid).
3. **تطبيق المصادقة** (TOTP - Time-based One-Time Password).
4. **بصمة الإصبع/الوجه** (إذا كان الجهاز يدعمها).

#### **كود إعداد TOTP (Node.js + speakeasy)**
```javascript
const speakeasy = require('speakeasy');
const QRCode = require('qrcode');

// توليد مفتاح سري للمستخدم
const secret = speakeasy.generateSecret({ length: 20 });

// إنشاء رابط QR لتطبيق المصادقة (مثل Google Authenticator)
const otpauthUrl = speakeasy.otpauthURL({
    secret: secret.base32,
    label: `MyApp:${user.username}`,
    issuer: 'MyApp'
});

// تحويل الرابط إلى صورة QR
QRCode.toDataURL(otpauthUrl, (err, dataUrl) => {
    console.log(dataUrl); // إرسال الصورة إلى واجهة المستخدم
});

// حفظ المفتاح السري في قاعدة البيانات
await usersDB.updateOne(
    { username: user.username },
    { $set: { '2fa.secret': secret.base32 } }
);
```

### **2.3 عملية المصادقة**
1. بعد التحقق الأولي من كلمة المرور، يُطلب من المستخدم إدخال رمز 2FA.
2. الخادم يتحقق من الرمز المدخل باستخدام المفتاح السري المخزن.

#### **كود التحقق من رمز TOTP**
```javascript
async function verify2FACode(username, token) {
    const user = await usersDB.findOne({ username });
    if (!user || !user['2fa'] || !user['2fa'].secret) {
        throw new Error('2FA not configured');
    }

    const verified = speakeasy.totp.verify({
        secret: user['2fa'].secret,
        encoding: 'base32',
        token: token,
        window: 1 // يسمح بخطأ ±30 ثانية
    });

    return verified;
}
```

## **3. إجراءات الاستعادة**
### **3.1 رموز الاستعادة**
- عند إعداد 2FA، يتم توليد مجموعة من رموز الاستعادة وحفظها في قاعدة البيانات.
- يمكن للمستخدم استخدام أحد هذه الرموز لاستعادة الوصول في حالة فقدان الجهاز.

#### **كود توليد رموز الاستعادة**
```javascript
const crypto = require('crypto');

function generateRecoveryCodes() {
    const codes = [];
    for (let i = 0; i < 5; i++) {
        codes.push(crypto.randomBytes(4).toString('hex').toUpperCase());
    }
    return codes;
}

// حفظ الرموز في قاعدة البيانات
await usersDB.updateOne(
    { username: user.username },
    { $set: { '2fa.recoveryCodes': codes } }
);
```

### **3.2 استعادة الحساب باستخدام رمز الاستعادة**
```javascript
async function verifyRecoveryCode(username, recoveryCode) {
    const user = await usersDB.findOne({ username });
    if (!user || !user['2fa'] || !user['2fa'].recoveryCodes) {
        throw new Error('Recovery codes not found');
    }

    const index = user['2fa'].recoveryCodes.indexOf(recoveryCode);
    if (index === -1) {
        throw new Error('Invalid recovery code');
    }

    // إزالة الرمز المستخدم
    user['2fa'].recoveryCodes.splice(index, 1);
    await usersDB.updateOne(
        { username: user.username },
        { $set: { '2fa.recoveryCodes': user['2fa'].recoveryCodes } }
    );

    return true;
}
```

### **4.2 تذكر الجهاز**

```javascript
const jwt = require('jsonwebtoken');

function generateRememberToken(userId) {
    return jwt.sign({ userId }, process.env.JWT_SECRET, { expiresIn: '30d' });
}

function verifyRememberToken(token) {
    try {
        const decoded = jwt.verify(token, process.env.JWT_SECRET);
        return decoded.userId;
    } catch (err) {
        return null;
    }
}
```

## **9. مثال كامل لتدفق تسجيل الدخول مع 2FA**
```javascript
// 1. تسجيل الدخول الأولي
app.post('/login', async (req, res) => {
    const { username, password } = req.body;
    const isValid = await verifyPassword(username, password);
    if (!isValid) {
        return res.status(401).json({ error: 'Invalid credentials' });
    }

    // 2. التحقق من تفعيل 2FA
    const user = await usersDB.findOne({ username });
    if (user['2fa'] && user['2fa'].enabled) {
        // إرسال رمز 2FA
        if (user['2fa'].method === 'sms') {
            const code = generate2FACode(); // توليد رمز عشوائي
            await sendSMSCode(user.phone, code);
            return res.json({ requires2FA: true, method: 'sms' });
        } else if (user['2fa'].method === 'totp') {
            return res.json({ requires2FA: true, method: 'totp' });
        }
```

# تصميم نظام مصادقة ثنائي العامل (2FA) لتطبيق ويب

## 1. تدفق البيانات:
1. **تسجيل الدخول الأولي:**
   - يدخل المستخدم اسم المستخدم وكلمة المرور.
   - يتحقق الخادم من صحة بيانات الدخول باستخدام خوارزمية تشفير قوية.
   - إذا كانت البيانات صحيحة، يتم إنشاء جلسة آمنة للمستخدم.

2. **إعداد المصادقة الثنائية (2FA):**
   - يُطلب من المستخدم اختيار طريقة 2FA: SMS، تطبيق مصادقة، بريد إلكتروني، أو بصمة الإصبع/الوجه.
   - يُطلب من المستخدم إعداد 2FA وفقًا للطريقة المختارة:
     - SMS/بريد إلكتروني: إدخال رقم الهاتف أو عنوان البريد الإلكتروني.
     - تطبيق مصادقة: مسح رمز QR وتنشيط الميزة.
     - بصمة الإصبع/الوجه: تنشيط الميزة على الجهاز.

3. **عملية المصادقة:**
   - عند محاولة الدخول، يُطلب من المستخدم إدخال اسم المستخدم وكلمة المرور.
   - بعد التحقق الأولي الناجح، يُرسل رمز 2FA إلى المستخدم عبر الطريقة المختارة.
   - يجب على المستخدم إدخال رمز 2FA لاستكمال المصادقة.
   - بعد المصادقة الناجحة، يمكن للمستخدم الوصول إلى حسابه وتطبيقاته.

## 2. إجراءات الاستعادة:
1. **النسخ الاحتياطي للبيانات:**
   - يقوم النظام بتخزين نسخة احتياطية من بيانات المستخدمين، بما في ذلك إعدادات 2FA، في مكان آمن ومُشفّر.

2. **استعادة الحساب:**
   - **رمز استعادة:** إرسال رمز استعادة إلى البريد الإلكتروني المسجل أو رقم الهاتف.
   - **أسئلة أمان:** إجراء التحقق عبر أسئلة أمان مسبقة التكوين.
   - **الدعم الفني:** التواصل مع الدعم الفني في حالة الحاجة إلى مساعدة إضافية.
   - **مفتاح استعادة:** توفير مفتاح استعادة يمكن تخزينه بآمان من قبل المستخدم.

## 3. ميزات إضافية:
1. **تذكر الجهاز:** خيار لتذكر الجهاز أو المتصفح لمدة معينة لتجنب طلب 2FA في كل مرة.
2. **استخدام بصمة الإصبع أو الوجه:** إذا كان الجهاز يدعمها، يمكن استخدامها كطريقة 2FA بديلة.
3. **إشعارات الأمان:** إرسال إشعارات إلى المستخدم عند كل محاولة دخول غير معتادة.
4. **حظر المحاولات الفاشلة:** حظر الحساب مؤقتًا بعد عدد معين من المحاولات الفاشلة للحماية من الهجمات القسرية.
5. **جدران الحماية:** استخدام جدران الحماية وتقنيات الكشف عن التهديدات لتعزيز حماية النظام.

## 4. خوارزميات الأمان المستخدمة:
1. **bcrypt:** لتشفير كلمات المرور.
2. **TOTP (Time-based One-Time Password):** لتوليد رموز 2FA المؤقتة.
3. **JWT (JSON Web Tokens):** لتذكر الجهاز.
4. **AES-256:** لتشفير البيانات الحساسة في قاعدة البيانات.

## 5. تدابير الأمان الإضافية:
1. **تشفير البيانات:** تشفير جميع البيانات الحساسة في قاعدة البيانات.
2. **HTTPS:** استخدام بروتوكول HTTPS لتأمين الاتصال بين العميل والخادم.
3. **CORS:** تقييد الوصول إلى واجهة برمجة التطبيقات من نطاقات محددة.
4. **مراقبة السجلات:** تسجيل جميع محاولات الدخول ومحاولات الاستعادة.
5. **التحديثات الأمنية:** تحديث المكتبات المستخدمة بانتظام لإصلاح الثغرات.

## 6. ملخص تدفق البيانات:
1. **تسجيل الدخول:**
   - المستخدم يدخل اسم المستخدم وكلمة المرور.
   - الخادم يتحقق من صحة البيانات.
   - إذا كانت صحيحة، يُطلب رمز 2FA.

2. **إدخال رمز 2FA:**
   - المستخدم يدخل الرمز المرسل عبر SMS/Email أو من تطبيق المصادقة.
   - الخادم يتحقق من الرمز.
   - إذا كان الرمز صحيحًا، يُسمح بالوصول.

3. **استعادة الحساب:**
   - المستخدم يطلب استعادة الحساب.
   - يُرسل رمز استعادة إلى البريد الإلكتروني أو الهاتف.
   - المستخدم يدخل الرمز لاستعادة الوصول.
